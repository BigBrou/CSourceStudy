
C++
    데이터를 묶고 처리하는 것을 어떤 분류에 따라 합치고 나눌 지에 대해 고민

C++ 에서 구조체
  - 구조체는 연관 있는 데이터를 묶을 수 있는 문법적 장치
  - 구조체 안에서 함수의 작성이 가능하다
    - 추가로 내부에서는 선언(Declaration)을 하고 외부로 빼서 (Scope 로 알려줌) 정의(Definition)할 수 있음
  - Enum 타입형도 추가가 가능

  외부의 함수에서 구조체를 parameter로 넣을 경우에는 참조자 (&) 를 사용해서 기입할 수 있음

C++ 클래스와 구조체의 차이
  - 구조체는 원래 public 에 대한 접근으로, 접근이 용이하다
  - 클래스는 접근을 못하게 막고 있어, 접근을 위해서는 별도의 접근과 관련된 지시 (protected, public) 선언을 따로 기입해야 함

C++ 클래스
  - 클래스는 객체를 생성 가능
  - 클래스 내에 선언된 변수를 맴버변수, 선언된 함수를 멤버함수라고 지칭

  - C++ 에서는 클래스 파일을 만들 때, header 파일과 cpp 파일 두 개를 생성하여 만들고,
    Header 파일에서는 선언(Declaration)에 대한 내용을
    CPP 파일에서는 정의(Definition)에 대한 내용을 기입한다.

  - 예외 case로 inline 함수가 존재하는데 이는 inline 함수는 컴파일 시에 함수의 내용도 함께 포함되어야 하기 때문에
    Header 파일에 넣음
